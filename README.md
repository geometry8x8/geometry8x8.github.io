# Геометрия — сайт

Статический сайт-визитка. Тексты и данные отделены от вёрстки: шаблон
наполняется словарями при сборке (build-time), а не во время запроса
(GitHub Pages не исполняет PHP/серверный код — только готовые файлы).

## Структура

```
src/
  templates/page.template.html   HTML-скелет с {{плейсхолдерами}}, без текста
  locales/{ru,kg,en}.json        словари — весь текст сайта
  assets/css/style.css           стили, включая токены светлой/тёмной темы
  assets/js/theme.js             клик по переключателю темы
  assets/img/logo.png            логотип (светлая тема)
  assets/img/logo-white.png      логотип (тёмная тема)
build.js                         сборщик: шаблон + словари -> dist/
.github/workflows/deploy.yml     сборка и деплой на GitHub Pages при push в main
CNAME                            geometry.kg
```

## Светлая/тёмная тема

Определение темы при первой загрузке — инлайн-скрипт в `<head>` шаблона,
выполняется до отрисовки (иначе будет вспышка неверной темы):

1. Есть сохранённая в `localStorage` тема → используется она.
2. Иначе — берётся системная настройка (`prefers-color-scheme`).

Клик по кнопке переключателя — `assets/js/theme.js`, сохраняет выбор в
`localStorage`; дальше приоритет всегда за явным выбором пользователя,
даже если он поменяет системную тему.

Все цвета — CSS-переменные в `style.css` (`:root` — светлая,
`[data-theme="dark"]` — тёмная). Корректировки оттенков — только в этих
двух блоках, остальной CSS ссылается на переменные.

Карта покрытия сети в этой версии отключена (фокус — идентификация
компании, планы, контакты). Логика демо-точек и SVG-сетки удалена из
`build.js`; при необходимости поднять обратно — точка отсчёта:
архив `geometriya-site-repo.zip` из предыдущей версии в этом чате.

## Локальная сборка и проверка

```
node build.js
```

Результат — папка `dist/` с тремя страницами (`/`, `/kg/`, `/en/`) и
скопированными assets. `dist/` не коммитится (см. `.gitignore`) —
собирается каждый раз заново, в том числе автоматически в CI.

**Важно: не открывай `dist/index.html` двойным щелчком (`file://...`).**
Пути к ресурсам в шаблоне абсолютные (`/assets/css/style.css`) — это
правильно для продакшена (GitHub Pages отдаёт сайт с корня домена), но
под `file://` браузер резолвит `/` в корень диска, а не в папку сайта —
CSS и картинки не найдутся, хотя сборка прошла без ошибок.

Для локальной проверки — маленький сервер без внешних зависимостей:

```
npm run preview
```

Соберёт `dist/` и поднимет `http://localhost:5000/` — открывать в
браузере оттуда, не из проводника. `Ctrl+C` — остановить.

## Первый деплой на GitHub

1. Создать пустой репозиторий на GitHub (например `geometriya-site`).
2. В этой папке:
   ```
   git init
   git add .
   git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/<твой-аккаунт>/geometriya-site.git
   git push -u origin main
   ```
3. В репозитории: **Settings → Pages → Build and deployment → Source:
   GitHub Actions**. Ничего собирать вручную не нужно — Action в
   `.github/workflows/deploy.yml` соберёт сайт и задеплоит при каждом
   push в `main`.
4. Там же, в Settings → Pages → Custom domain, указать `geometry.kg`
   (файл `CNAME` в корне репозитория это дублирует — GitHub подхватит
   его автоматически при первой сборке, но явное указание в интерфейсе
   надёжнее).
5. У регистратора домена `geometry.kg` прописать DNS:
   - Тип `A`, четыре записи на IP GitHub Pages:
     `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - (опционально, если понадобится `www.geometry.kg`) `CNAME` на
     `<твой-аккаунт>.github.io`
6. Включить "Enforce HTTPS" в Settings → Pages после того как DNS
   применится (обычно от нескольких минут до пары часов).

## Как редактировать контент

- Текст — только в `src/locales/*.json`. Ключи одинаковые во всех трёх
  файлах; если добавляешь ключ — добавь во все три, иначе сборка
  выведет предупреждение `[build] нет значения для ключа {{...}}`
  и на странице останется видимый `{{ключ}}`.
- Точки карты — `src/data/points.json`. При появлении реальных
  координат из реестра экранов — заменить x/y (сейчас это условные
  координаты на стилизованной сетке, не GPS) на реальную привязку —
  это отдельная задача (см. риск ниже).
- Вёрстка/стили — `src/templates/page.template.html` и
  `src/assets/css/style.css`, не трогая словари.

После любого изменения — просто `git push`, деплой запустится сам.

## Динамика и адаптив

- `assets/js/motion.js` — три поведения на `IntersectionObserver`:
  1. **Высота шапки → `--header-h`**: считается один раз при загрузке и при
     ресайзе, используется и в CSS (`scroll-margin-top` секций — чтобы
     переход по якорной ссылке не прятал контент под sticky-шапкой), и в
     самом наблюдателе (`rootMargin`) — единая цифра, а не два независимых
     подобранных на глаз значения.
  2. **Scroll-spy**: треугольник-индикатор раздела (`data-spy="..."`)
     пульсирует у секции с наибольшим текущим процентом видимости во
     вьюпорте (`intersectionRatio`), не у ближайшей по краю. Добавить новый
     раздел в цепочку — дать секции `id`, а триггеру `data-spy="тот-же-id"`.
  3. **Reveal**: элементы `.reveal` плавно появляются при входе в вьюпорт и
     плавно скрываются при выходе — в обе стороны, при каждом прохождении
     (не одноразово). Анимация объявлена в CSS только внутри
     `@media (prefers-reduced-motion: no-preference)`.
- `assets/js/nav.js` — бургер-меню на брейкпоинте `≤860px` вместо переноса
  навигации второй строкой (не подходило для разной длины меню на трёх
  языках). Клавиатура (Escape), клик вне области, автозакрытие при ресайзе
  обратно на десктоп — всё учтено.
- 3D — сознательно не трогается в этой итерации (отдельное решение позже).
- Брейкпоинты: `1024px` (2 колонки в карточках), `860px` (бургер вместо
  навигации, одноколоночные сетки), `480px` (компактная типографика,
  полноширинные кнопки, увеличенные touch-таргеты).

## Известные ограничения (перенесены из предыдущей версии, не сняты)

- Email и телефон в контактах — плейсхолдеры, заменить в каждом из
  трёх `locales/*.json` (ключи не общие — специально, чтобы номер
  можно было при необходимости различать по локали, но email/phone
  сейчас одинаковые везде, менять во всех трёх файлах синхронно).
- Точки на карте — демо-данные, не реальные координаты сети.
- Перевод KG выполнен не носителем языка, требует вычитки перед
  публикацией.
- Целевые показатели стратегии (500 носителей / 36 месяцев) намеренно
  не публикуются — см. обоснование в предыдущем разборе (конкурентная
  экспозиция перед Foel).
